# Generator Output Schema Rules

Every generator in this directory produces JSON or YAML output that is consumed by
the researchartifacts.github.io website and documented in researchartifacts/data-schemas.

**When changing output fields:**
1. Update the matching schema in `../../data-schemas/schemas/` (see mapping in `.github/copilot-instructions.md`).
2. Keep `additionalProperties: false` in schemas — any new field must be explicitly added.
3. If adding an optional field, do NOT add it to the schema's `required` array.
4. If changing a field type (e.g., string → array), use `oneOf` in the schema.
5. After changes, validate: `cd ../../data-schemas && .venv/bin/python -c "import json,jsonschema; s=json.load(open('schemas/SCHEMA.schema.json')); jsonschema.Draft202012Validator.check_schema(s); print('OK')"`
